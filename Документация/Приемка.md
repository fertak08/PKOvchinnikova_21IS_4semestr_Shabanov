## **3. Функциональные требования (проверка по ТЗ и бизнес-требованиям)**

| № | Функция | Метод проверки | Статус |
|---|---------|----------------|--------|
| 1 | Регистрация и аутентификация пользователей | Проверить регистрацию преподавателя и студента | |
| 2 | Разделение ролей: преподаватель / студент | Проверить доступ к разным функциям в зависимости от роли | |
| 3 | Создание задания преподавателем | Создать задание с описанием, сроками, материалами | |
| 4 | Определение критериев оценки для задания | Добавить 3+ критерия (бинарные/градуированные) | |
| 5 | Уведомление студентов о новом задании | Проверить уведомление в системе или через Telegram | |
| 6 | Загрузка работы студентом | Студент загружает файл в систему до дедлайна | |
| 7 | Автоматическая проверка по критериям | Система подсчитывает выполненные критерии | |
| 8 | Автоматическое определение оценки по матрице | Проверить оценки 5, 4, 3, 2 в зависимости от критериев | |
| 9 | Ручная корректировка оценки преподавателем | Преподаватель меняет оценку, добавляет комментарий | |
| 10| Возврат работы на доработку | Преподаватель возвращает работу студенту с комментариями | |
| 11| Пересдача работы | Студент загружает исправленную версию | |
| 12| Просмотр журнала успеваемости | Студент видит свои оценки, прогресс | |
